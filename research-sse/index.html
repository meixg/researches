Server-Sent Events (SSE)

一种简单、强大的通过 HTTP 进行实时更新流传输的方式。

什么是 SSE?

SSE 是一种允许服务器通过 HTTP 向网页推送数据的标准。与 WebSockets 不同，它是单向的（服务器 → 客户端）。

主要优势:

  • 基于 HTTP: 无需特殊协议或服务器。兼容大多数代理和防火墙。
  • 自动重连: 如果连接中断，浏览器会自动尝试重新连接。
  • 轻量级: 与轮询或全双工 WebSockets 相比，开销较低。
工作原理: 服务器发送 Content-Type: text/event-stream 响应头并保持连接开启。

协议格式:

id: 123
event: notification
data: Hello World!
retry: 5000

: 这是一条注释

JavaScript API:

const source = new EventSource('/events');

source.onmessage = (e) => {
  console.log('新消息:', e.data);
};

source.addEventListener('notification', (e) => {
  console.log('自定义事件:', e.data);
});
