AG-UI 协议调研报告

标准化 AI Agent 与前端应用的交互协议 (Agent User Interaction Protocol)

1 什么是 AG-UI？

AG-UI 是一套开放、轻量、基于事件的协议，旨在规范 AI 智能体（Agent）如何连接到用户界面。它不仅是简单的请求/响应，更是为长时运行、非确定性、多模态的 Agent 交互量身定做的。

  • 流式响应: 实时传输文本 Token 和执行事件。
  • 状态同步: 保持前端 UI 与后端 Agent 状态一致。
  • 工具透明: 实时展示工具调用过程及结果。
  • 人机协作: 支持执行中断、审批和恢复。

架构模型

[ 用户界面 (React/Vue/CLI) ]
      |
      | HTTP POST (指令/消息)
      V
[ AG-UI 协议层 ] <--- (SSE 流式事件回调)
      |
      V
[ Agent 运行时 (LangGraph/CrewAI/...) ]

2 通信机制：HTTP POST + SSE

客户端发起请求 (HTTP POST)
POST /agent/run
Content-Type: application/json
Accept: text/event-stream

{
    "threadId": "conv_123",
    "messages": [{ "role": "user", "content": "帮我查询明天的天气" }]
}
服务器流式返回 (SSE)
data: {"type": "RUN_STARTED", "runId": "run_001", "threadId": "conv_123"}

data: {"type": "TEXT_MESSAGE_START", "messageId": "msg_001", "role": "assistant"}

data: {"type": "TEXT_MESSAGE_CONTENT", "messageId": "msg_001", "delta": "正在"}

...

data: {"type": "RUN_FINISHED", "runId": "run_001"}

3 核心事件详解

Lifecycle (生命周期)

管理 Agent 运行状态。

RUN_STARTED: 建立 runId 作用域。
STEP_STARTED: 标记子步骤开始。
RUN_FINISHED: 运行成功结束。

Streaming (文本流)

逐 Token 显示消息内容。

TEXT_MESSAGE_START: 初始化消息气泡。
TEXT_MESSAGE_CONTENT: delta 字段含增量。
TEXT_MESSAGE_END: 消息完成。

Tool Call (工具调用)

透明展示 Agent 的思考与动作。

TOOL_CALL_START: 显示调用的工具名。
TOOL_CALL_ARGS: 流式展示参数 JSON。
TOOL_CALL_RESULT: 返回工具执行结果。

State (状态更新)

同步复杂的 Agent 内部状态。

STATE_SNAPSHOT: 全量同步。
STATE_DELTA: JSON Patch 增量同步。

4 示例代码实现

Python (FastAPI) 服务器实现

from fastapi import FastAPI, Request
from fastapi.responses import StreamingResponse
import json, asyncio

app = FastAPI()

async def event_generator():
    # 模拟 AG-UI 事件流
    yield f"data: {json.dumps({'type': 'RUN_STARTED', 'runId': '123'})}\n\n"
    await asyncio.sleep(1)

    yield f"data: {json.dumps({'type': 'TEXT_MESSAGE_START', 'messageId': 'm1', 'role': 'assistant'})}\n\n"

    for word in ["你好", "，我是", "AI", "助手"]:
        yield f"data: {json.dumps({'type': 'TEXT_MESSAGE_CONTENT', 'messageId': 'm1', 'delta': word})}\n\n"
        await asyncio.sleep(0.5)

    yield f"data: {json.dumps({'type': 'TEXT_MESSAGE_END', 'messageId': 'm1'})}\n\n"
    yield f"data: {json.dumps({'type': 'RUN_FINISHED', 'runId': '123'})}\n\n"

@app.post("/agent/run")
async def run_agent(request: Request):
    return StreamingResponse(event_generator(), media_type="text/event-stream")

JavaScript 前端消费实现

async function startChat(userInput) {
  const response = await fetch('/agent/run', {
    method: 'POST',
    body: JSON.stringify({ messages: [{ role: 'user', content: userInput }] }),
    headers: { 'Accept': 'text/event-stream' }
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { value, done } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n');

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const event = JSON.parse(line.substring(6));
        handleAGUIEvent(event);
      }
    }
  }
}

function handleAGUIEvent(event) {
  switch(event.type) {
    case 'TEXT_MESSAGE_CONTENT':
      console.log('收到 Token:', event.delta);
      // 更新 UI 逻辑
      break;
    case 'RUN_FINISHED':
      console.log('执行结束');
      break;
  }
}

5 协议扩展性 (Extensibility)

AG-UI 并非封闭协议，它通过以下三种核心机制支持自定义扩展，确保能适配各种垂直领域的需求。

Custom Events

最常用的扩展方式，用于应用特定的指令。

{
  "type": "CUSTOM",
  "name": "SHOW_CHART",
  "value": { "type": "pie", "data": [10, 20] }
}

Raw Events

用于透传外部协议（如 MCP）的原始载荷。

{
  "type": "RAW",
  "source": "mcp-server",
  "event": { ...mcpPayload }
}

Meta Events

用于独立于运行状态的元数据（草案）。

{
  "type": "META",
  "metaType": "thumbs_up",
  "payload": { "messageId": "m1" }
}
